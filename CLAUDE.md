# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

The **Egon Zippel Polaroid Archive Search** — a static site that lets visitors search ~8,700 daily polaroid sketches by Egon Zippel (1989–present). The archive is mirrored from `egonzippel.com`, enriched with Gemini-generated OCR / visual descriptions / keywords, and served as a single JSON blob that a Lunr.js client indexes in the browser.

The project has three logical pieces that all share the same metadata schema:

1. **Frontend** (`index.html`, `style.css`, `search_logic.js`) — static site, no build step. Loads `image_metadata_search.json` directly from R2 and builds a Lunr index in the browser.
2. **Local Python pipeline** (`scrape_polaroids.py`, `populate_ai_data.py`) — original/manual ingestion path. Scrapes the source site to CSV, downloads images, runs Gemini analysis, writes `image_metadata.json`.
3. **Cloudflare Worker pipeline** (`worker/`) — daily cron-triggered re-implementation of the same scrape → analyze → export pipeline, but reading/writing metadata in R2 instead of local files.

The two pipelines are **alternates, not complements** — both produce the same `image_metadata.json` shape. The Worker is the live system; the Python scripts are kept for one-off backfills and local experimentation.

## Data flow & file roles

```
egonzippel.com  ──scrape──▶  polaroids_data.csv  ──download──▶  scraped_images/*.jpg
                                                                       │
                                                                       ▼
                                                                Gemini (OCR + caption + keywords)
                                                                       │
                                                                       ▼
                                                          image_metadata.json   (full, source of truth)
                                                                       │
                                                              trim + gzip
                                                                       ▼
                                              image_metadata_search.json(.gz)   (served to frontend via R2)
```

- `image_metadata.json` — full record per image, **gitignored** (see `.gitignore`). Source of truth.
- `image_metadata_search.json` — frontend subset (drops `local_path`, items with failed analysis). Generated by `worker/src/exporter.js`.
- `image_metadata_compact.json` — older trimmed variant; check git history before assuming a producer.
- The frontend hard-codes the R2 public URL in `search_logic.js` (`DATA_URL`). When the bucket / public URL changes, update that constant.

## Metadata schema

```json
{
  "filename": "2011-04-15.jpg",
  "year": "2011", "month": "04", "day": "15",
  "suffix_original": "", "sortable_suffix": "0", "extension": "jpg",
  "local_path": "scraped_images/2011-04-15.jpg",
  "image_url":     "https://...full-size on R2 / egonzippel CDN...",
  "thumbnail_url": "https://...thumbnail...",
  "source_page":   "https://egonzippel.com/polaroids/2011/1/<index>",
  "ai_analysis": {
    "ocr_text": "waiting for the train",
    "visual_description": "A sketch of a person waiting at a train platform",
    "keywords": ["train", "waiting", "platform", "sketch"]
  }
}
```

`source_page` is constructed as `{year_url}/1/{img_index}` where `img_index` is the position in that year's gallery — the trailing index matters; do not strip it.

`ai_analysis` is `null` until processed, and on failure becomes `{"error": "...", ...}`. The exporter filters out items where `ai_analysis.error` is set, so failed items never reach the frontend.

## Commands

### Python deps (uv-managed)
```bash
uv sync                         # install from pyproject.toml / uv.lock
uv run python scrape_polaroids.py
uv run python populate_ai_data.py    # requires GEMINI_API_KEY (.env loaded via python-dotenv)
```
Runtime is Python 3.13 (see `pyproject.toml`). Note: `populate_ai_data.py` uses the **OpenAI SDK pointed at Gemini's OpenAI-compatible endpoint** (`generativelanguage.googleapis.com/v1beta/openai/`), not the native Gemini SDK — keep that in mind when changing API calls.

### Frontend
```bash
python -m http.server 8000      # any static server works; opens at /index.html
```
No build step. The page fetches `image_metadata_search.json` from R2, so it works the same locally and on GitHub Pages.

### Cloudflare Worker
```bash
cd worker
npm install
npm run dev          # wrangler dev — local
npm run deploy       # wrangler deploy
npm run tail         # stream production logs
# Manual trigger in prod: GET / on the deployed worker URL (optionally ?limit=N to cap analyzed images)
# Health check:           GET /health
```
Cron is daily at 06:00 UTC (`worker/wrangler.toml`). The worker reads/writes `image_metadata.json` in R2 bucket `egon-image-metadata` (binding `METADATA_BUCKET`), and writes `image_metadata_search.json{,.gz}` alongside it.

## Search behavior (`search_logic.js`)

- Lunr fields with explicit boosts: `keywords_string` (15), `ocr_text` (10), `filename` (5), `visual_description` (2). Changing these meaningfully changes ranking.
- Smart fuzzy: words **8+ letters** automatically get edit-distance-1 fuzzy matching; shorter words use exact match. Manual `~` works too. The threshold was bumped from 6 to avoid false positives like `claude`→`cloud`.
- Quoted phrases bypass Lunr and use a manual case-insensitive substring match across the same fields.
- Date tokens (4-digit years, full month names, 3-letter month abbreviations) are extracted from the query before Lunr sees it and applied as filters.
- Images render directly from `thumbnail_url` / `image_url`. The old CORS-proxy extraction via `api.allorigins.win` was removed in `fb08c3e` — do not reintroduce it; URLs in the metadata are already direct.

## Conventions worth knowing

- The artist signs as **"Egon"**, **"Egon Zippel"**, or **"Egon NYC"** — the Gemini prompt in `populate_ai_data.py` and `worker/src/analyzer.js` calls this out for OCR. Keep both prompts in sync if you tweak one.
- Filenames are `YYYY-MM-DD[suffix].jpg` where multiple polaroids on the same day get a suffix (e.g. `2011-04-15B.jpg`, `2011-04-15.2.jpg`). Sort order uses `sortable_suffix` derived in `parse_filename` — don't sort by raw filename.
- The CSV (`polaroids_data.csv`) and `scraped_images/` are intermediate artifacts of the local pipeline only. The Worker pipeline doesn't produce them.
- `raw_data/`, `scraped_images/`, `image_metadata.json`, `*.env`, `.wrangler/`, and `worker/node_modules/` are all gitignored — don't commit them even if asked to "commit everything".
